 # 🚀 上架步驟（最簡單方式）

## ⚠️ 如果遇到「空倉庫」錯誤

如果看到錯誤：`The provided GitHub repository does not contain the requested branch or commit reference`

**解決方法：**
1. 雙擊執行 `首次推送.bat`（專門處理空倉庫問題）
2. 或參考 `解決空倉庫問題.md` 的詳細說明

---

## 方式 1: 使用 Vercel（推薦，5 分鐘完成）

### 步驟 1: 準備 GitHub 倉庫

1. **在 GitHub 建立新倉庫**
   - 前往：https://github.com/new
   - 倉庫名稱：`marine-ai-assistant`
   - 選擇 Public 或 Private
   - 點擊 "Create repository"

2. **設定 GitHub 認證**（第一次需要）
   
   **方法 A：使用批次腳本（最簡單）**
   - 雙擊執行 `setup-github-auth.bat`
   - 或雙擊執行 `快速推送.bat`（會自動引導）
   
   **方法 B：手動設定 Personal Access Token**
   - 前往：https://github.com/settings/tokens
   - 點擊 "Generate new token" → "Generate new token (classic)"
   - 勾選 `repo` 權限
   - 複製產生的 token（格式：`ghp_xxxxxxxxxxxxx`）
   - 推送時，密碼欄位貼上這個 token（不是 GitHub 密碼！）

3. **推送專案到 GitHub**
   
   **方法 A：使用批次腳本**
   ```batch
   快速推送.bat
   ```
   
   **方法 B：手動執行**
   ```bash
   git init
   git add .
   git commit -m "準備部署"
   git branch -M main
   git remote add origin https://github.com/jjjtyy666/marine-ai-assistant.git
   git push -u origin main
   ```
   
   ⚠️ **推送時如果要求輸入密碼**：
   - Username: `jjjtyy666`
   - Password: 貼上 Personal Access Token（不是 GitHub 密碼！）

   📖 **詳細認證說明**：請參考 `設定GitHub認證.md`

### 步驟 2: 在 Vercel 部署

1. **訪問 Vercel**
   - 前往：https://vercel.com
   - 使用 GitHub 帳號登入

2. **導入專案**
   - 點擊 "Add New..." → "Project"
   - 選擇您的 GitHub 倉庫
   - 點擊 "Import"

3. **設定環境變數**
   - 在專案設定頁面，點擊 "Environment Variables"
   - 添加以下變數：

```
VITE_AI_PROVIDER = deepseek
VITE_DEEPSEEK_IS_LOCAL = false
VITE_DEEPSEEK_API_KEY = 您的DeepSeek_API_Key
VITE_USE_REAL_WEATHER = true
VITE_USE_REAL_SEA_STATE = true
VITE_WEATHER_PROVIDER = cwb
VITE_SEA_STATE_PROVIDER = cwb
VITE_CWB_API_KEY = CWA-265A4DB1-FD45-4491-971F-D0D6B92DB008
VITE_USE_MOCK_DATA = false
```

4. **部署**
   - 點擊 "Deploy"
   - 等待建置完成

5. **完成！**
   - 獲得部署 URL，例如：`https://your-project.vercel.app`

## ⚠️ 重要提醒

### 關於本地 AI 服務

如果使用本地 Ollama，部署後無法訪問 `localhost:11434`。

**解決方案**：
- **選項 A**：使用 DeepSeek 官方 API
  - 設定 `VITE_DEEPSEEK_IS_LOCAL=false`
  - 設定 `VITE_DEEPSEEK_API_KEY=您的API_Key`

- **選項 B**：暫時使用 Mock 資料
  - 設定 `VITE_USE_MOCK_DATA=true`
  - 之後再切換到真實 API
